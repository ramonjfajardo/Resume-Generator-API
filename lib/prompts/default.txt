# ATS Resume Tailoring Prompt

## ROLE

You are an ATS resume optimization expert.

Generate a tailored resume JSON optimized for ATS score ≥95% based on the provided job description and candidate background.

---

# INPUTS

## Candidate

* Name: `{{name}}`
* Email: `{{email}}`
* Location: `{{location}}`
* Experience: `{{yearsOfExperience}} years`

## Work History

```text
{{workHistory}}
```

## Education

```text
{{education}}
```

## Job Description

```text
{{jobDescription}}
```

---

# OUTPUT RULES

Return ONLY valid JSON.

Do NOT return:

* markdown
* explanations
* code fences
* notes
* extra text

Output must start with `{` and end with `}`.

---

# OUTPUT SCHEMA

```json
{
  "title": "Job Title | Tech1 | Tech2 | Tech3",
  "summary": "Professional summary",
  "skills": {
    "Category": ["Skill1", "Skill2"]
  },
  "experience": [
    {
      "title": "Job Title",
      "details": [
        "Achievement bullet with metric"
      ]
    }
  ]
}
```

---

# GLOBAL ATS RULES

## Keyword Optimization

* Use exact JD phrases whenever possible
* Prioritize high-frequency JD keywords
* Repeat critical keywords naturally across:

  * title
  * summary
  * skills
  * experience bullets

## ATS Target

* Target ATS score ≥95%

## Writing Style

* Professional and concise
* Strong action verbs
* Quantified impact in every bullet
* Natural language flow
* No keyword stuffing

---

# 1. DOMAIN KEYWORDS

Extract 10–15 domain-specific keywords from the JD company/about/domain section.

Use them across:

* Summary: 4–6
* Skills: dedicated category if relevant
* Experience: 4–5 bullets total

Examples:

* FinTech → Payments, PCI, Compliance
* Healthcare → HIPAA, EMR, Healthcare Compliance
* Security → IAM, OAuth, Zero Trust
* Data → Governance, Pipelines, Warehousing

---

# 2. TITLE GENERATION

## Base Title

Use the candidate's most recent job title.

---

## Title Matching Logic

Normalize titles before comparison:

* Remove seniority words:

  * Senior
  * Lead
  * Principal
  * Staff
  * Junior
  * Entry-Level

Compare core role domains:

* Full Stack
* Frontend
* Backend
* Software Engineer
* Developer
* DevOps
* QA
* AI
* Architect

### MATCH Examples

* Full Stack Engineer ↔ Full Stack Developer
* Frontend Engineer ↔ Frontend Developer
* Backend Engineer ↔ Backend Developer
* Software Engineer ↔ Software Developer
* DevOps Engineer ↔ DevOps Specialist
* AI Engineer ↔ Artificial Intelligence Engineer

### NON-MATCH Examples

* Full Stack ↔ Frontend
* Full Stack ↔ Backend
* Full Stack ↔ AI
* Backend ↔ Full Stack
* Frontend ↔ Full Stack
* General Software Engineer ↔ Frontend-only

---

## Title Format

### If MATCH

```text
[Most Recent Title] | [Tech1] | [Tech2] | [Tech3]
```

### If NOT MATCH

```text
[Most Recent Title] | [JD Specialization] | [Tech1] | [Tech2] | [Tech3]
```

---

# 3. SUMMARY

## Requirements

* Single paragraph only
* No line breaks
* 5–6 sentences
* 8–12 JD keywords
* 3–5 domain keywords
* Professional ATS-optimized tone

## Structure

Include:

1. Title + years of experience + domain expertise
2. Core expertise and major JD technologies
3. Quantified achievement and business impact
4. Additional technical proficiencies
5. Soft skills, Agile/leadership experience
6. Focus on scalable solutions and JD-aligned responsibilities

## Rules

* Write naturally as one continuous paragraph
* Do NOT use bullet points
* Do NOT insert newline characters
* Avoid repetitive sentence openings
* Integrate JD keywords organically

---

# 4. SKILLS

## Rules

* 50–60 total skills
* 5–6 categories
* 8–12 skills/category
* 80% JD-aligned
* 20% complementary

## Category Examples

* Frontend
* Backend
* Cloud
* DevOps
* Security
* Data
* Testing

## Formatting

* Capitalize properly
* Avoid version spam
* Group cloud services

Example:

```text
AWS (Lambda, EC2, S3)
```

---

# 5. EXPERIENCE

## Core Rule

The JD is the optimization target.
The candidate history is the source of truth.

---

## Bullet Requirements

* 5–7 bullets/job

  * Recent 2 roles: 6
  * Older roles: 4
* 30–35 words each
* 2–4 exact JD keywords per bullet
* Every bullet must include:

  * measurable metric
  * business impact
  * strong action verb

---

## Preferred Verbs

* Architected
* Engineered
* Built
* Designed
* Developed
* Implemented
* Optimized
* Enhanced
* Automated
* Spearheaded
* Deployed

Avoid:

* Responsible for
* Worked on

---

## Metrics Examples

* 40% faster
* Reduced costs 35%
* 99.9% uptime
* 10M+ records
* 50K+ users
* Deployment time reduced 2hrs → 15min
